# 03 HTTP 메세지 ~ 04 커넥션 관리

## 문제
파이프라인 커넥션에 대한 설명 중 가장 정확한 것을 고르세요.

1. 파이프라인 커넥션은 응답을 기다리지 않고 여러 HTTP 요청을 연속으로 보내어 왕복 지연 시간을 단축한다.그러나 HTTP 응답은 요청 순서와 무관하게 도착하며, 클라이언트는 이를 재정렬 처리해야 한다.
2. 클라이언트는 커넥션이 지속 커넥션인지 확인하는 절차 없이 즉시 요청을 파이프라이닝해야 성능 향상을 극대화하며, 응답은 도착하기 전에 큐에 쌓여 전송된다.
3. 파이프라인 커넥션의 사용은 POST 요청과 같이 서버에 여러 번 실행될 경우 문제가 발생하는 비멱등 요청에 한해서만 허용된다.
4. 파이프라인 커넥션은 여러 개의 요청을 응답이 도착하기 전에 큐에 쌓아 보낼 수 있게 하여 대기 시간이 긴 네트워크 상황에서 성능을 향상시킨다.
5. 클라이언트는 커넥션이 끊어졌다는 예상치 못한 상황에 직면하더라도, 서버 부하 방지를 위해 파이프라인에 남아있는 미완료 요청들은 재전송해서는 안 된다.

## 정답

3번

## 해설
1 -> 응답은 요청 순서를 반드시 따라야 한다.
2 -> 지속 커넥션인지 우선 확인을 해야 한다.
3 -> 비멱등 요청은 파이프라인을 사용하면 안된다.
5 -> 멱등 요청의 경우는 재전송이 가능하다.

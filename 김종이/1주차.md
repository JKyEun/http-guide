#### 01. HTTP 개관 ~ 02. URL과 리소스

##### 📘 문제

> 다음 중 틀린 설명을 고르시오.

1. URL은 두가지 주요 부분집합인, URL과 URN으로 구성된 종합적인 개념이다.
2. URL은 현재 그 리소스가 어디에 존재하든 상관없이 그 이름만으로 리소스를 식별한다.
3. HTTP 명세에서는 URI을 더 일반화된 개념의 리소스 식별자로 사용하며, URL은 URI의 한 부분으로 취급된다.
4. URL은 브라우저가 정보를 찾는데 필요한 리소스 위치를 가리킨다.
5. URL을 사용해서 리소스를 일관된 방식으로 지칭할 수 있다.

##### 📘 1주차 문제 정답: 2번

##### 📘 문제 해설

- 1번 : URI ⊃ {URL, URN} 관계입니다.
  - URI는 리소스를 식별하는 문자열
  - URL은 위치(Location) 기반
  - URN은 이름(Name) 기반
- 2번 : URL은 리소스의 위치(Location)로 식별합니다.
- 3번 : HTTP는 URI를 일반 개념으로 사용합니다.
- 4번 : 위치를 가리키는 것이 URL의 핵심입니다.
- 5번 : URL를 사용하면 일관된 스킴(http://, https://, ftp:// 등)을 통해 동일한 형식으로 지칭 가능합니다.
